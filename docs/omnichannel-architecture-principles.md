# Omnichannel Architecture Principles

**One Trust Gate core. Many future surfaces. Same rules. Same receipts. Same safety boundary.**

## Principles

- Maintain one shared Trust Gate core.
- Allow many possible future discovery and integration surfaces.
- Do not duplicate decision logic per channel.
- Treat channels as interfaces, not separate products.
- Keep receipts and policy decisions consistent across every future interface.
- Carry the complete safety boundary with every future channel.
- Start with code launch and extend later only after review.
- Avoid rebuilding later by choosing adaptable foundations now.

The local code, manifest, schemas, examples, CLI, and capability statement are the current interfaces. MCP/A2A adapters, hosted services, package publication, payments, settlement, and external agent contact are not active.

AUC is not integrated. Agent Contact System is not integrated. Omnichannel design does not weaken the core rule:

**No mandate. No evidence. No verified intent. No signed gate pass. No settlement.**
