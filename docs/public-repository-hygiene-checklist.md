# Public Repository Hygiene Checklist

Use this checklist before publishing the local-first, `local_demo_only` code repository.

## Sensitive and operational material

- [ ] No real secrets.
- [ ] No tracked `.env` file.
- [ ] No real tokens.
- [ ] No real credentials.
- [ ] No database or database connection.
- [ ] No private logs or generated evidence records.
- [ ] No live endpoints.
- [ ] No payment rails or payment activation instructions.
- [ ] No wallet or banking logic or identifiers.
- [ ] No cloud calls or network calls.
- [ ] No external-agent contact or agent outreach automation.

## Public developer package

- [ ] README positioning and safety limits are clear.
- [ ] Manifest status and disabled capabilities are clear.
- [ ] Schema JSON files parse.
- [ ] Example JSON files parse and use synthetic data.
- [ ] `npm test` passes.
- [ ] `npm run build` passes.
- [ ] `npm run typecheck` passes.

## Separation and claims

- [ ] AUC is not integrated.
- [ ] Agent Contact System is not integrated.
- [ ] Quantum/post-quantum wording remains roadmap-only and makes no certification claim.
- [ ] Fast Reaction Trust Gate preserves the rule that speed never overrides trust.
- [ ] No production payment, legal, compliance, security, or official-standard claim is made.

**No mandate. No evidence. No verified intent. No signed gate pass. No settlement.**
