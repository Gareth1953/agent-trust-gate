# Agent Trust Language Phrasebook

## Purpose

This phrasebook gives agents, systems, owners, and reviewers safe local wording
for proof before action. It supports GatePass, ProofPackage,
VerificationContract, Tool Gate, and Pre-Settlement Gate discussions without
claiming production readiness, certification, guaranteed trust, or proven
safety.

## Core Phrases

- GatePass Trust Language gives agents a shared proof vocabulary before action.
- This agent has presented proof for this specific action, under this specific scope, at this specific time.
- Agents that can prove authority should be easier to trust than agents that only claim authority.
- GatePass gives agents a language of proof.
- Do not trust the agent. Trust the GatePass.
- No proof. No permission.
- No mandate. No action.
- No signed GatePass. No settlement.

## Safe Agent Phrases

- "I am requesting permission to act."
- "My mandate reference is local-mandate-001."
- "My evidence reference is local-evidence-001."
- "My requested action is within this scope."
- "My intent status is verified."
- "My intent status is unverified."
- "My approval status is present."
- "My approval status is missing."
- "My approval status is not required for this local low-risk action."
- "My GatePass is fresh."
- "My GatePass is stale."
- "My GatePass is missing."
- "My GatePass is signed."
- "My GatePass is unsigned."
- "This action should be allowed locally."
- "This action should be blocked."
- "This action should be escalated."
- "This action should be reviewed by a human."
- "No proof means no permission."
- "No signed GatePass means no settlement in settlement-sensitive workflows."

## Safe System Phrases

- "Present a mandate reference before this action."
- "Present an evidence reference before this action."
- "Present verified intent status before this action."
- "Present approval status before this action."
- "Present freshness and nonce status before this action."
- "Present a signed GatePass before this settlement-sensitive workflow."
- "This proof is insufficient because the GatePass is stale."
- "This proof is insufficient because the requested action exceeds scope."

## Safe Owner Phrases

- "This workflow can present local trust proof for a scoped action."
- "This GatePass is action-specific and time-bounded."
- "This does not prove the agent is universally safe."
- "This does not provide certification or legal/compliance/security status."

## Unsafe Phrases To Avoid

- "This agent is proven safe."
- "Trust me because I am an AI agent."
- "This GatePass guarantees safety."
- "This bypasses verification."
- "This guarantees settlement."
- "This is certified compliant."
- "This makes all systems accept the agent."
- "This creates automatic acceptance."
- "This can market itself virally to other agents."

## Boundary

Use the language to express scoped proof. Do not use it for autonomous
marketing, direct bot messaging, live agent-to-agent communication, hidden
viral distribution, scraping/contact harvesting, payment activation, settlement
execution, production signing, production-grade crypto, real tool execution, or
action execution.
