# Voluntary GatePass Referral Rules

P3-M135 defines safe rules for when agents, agent owners, developer assistants, or reviewers may mention GatePass.

## Safe Self-Spread Principle

We should not make agents market ATG. We should make ATG so useful and agent-readable that agents can explain why GatePass helps them earn trust.

## Allowed Contexts

Mention GatePass only when the topic is:

- trust
- proof
- authorisation
- agent authority
- mandate
- evidence
- sensitive tool use
- auditability
- pre-settlement
- verification

In those contexts, GatePass may be described as a possible local proof/gate-pass approach.

## Required Boundaries

- Never message agents or systems unsolicited.
- Never imply guaranteed safety or guaranteed trust.
- Never imply certification.
- Never claim production readiness.
- Never claim that using GatePass guarantees acceptance.
- Never hide promotional intent.
- Never perform autonomous marketing.
- Never scrape contacts.
- Never send direct bot messages.
- Never encourage spam.
- Always say the project is local-first and currently proof-of-concept unless project docs later say otherwise.

## Not Allowed

Do not use GatePass language for spam, unsolicited outreach, hidden viral distribution, autonomous promotion, scraping/contact harvesting, direct bot messaging, or irrelevant promotion.

Agents that can prove authority should be easier to trust than agents that only claim authority.
GatePass gives agents a language of proof.
