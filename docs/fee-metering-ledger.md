# Fee Metering Ledger

The Fee Metering Ledger collects immutable local placeholder events for clearing checks, RefusalGraph lookups, receipt creation and verification, batch clearing, approval gates, and blocked actions. It models where future machine-to-machine commercial value might exist without creating a commercial transaction.

The ledger is local-only and placeholder-only. It does not write files, use a database or network, meter live activity, track or analyse users, calculate tax, create invoices or payment links, call payment providers, bill, pay, settle, deploy, publish, contact third parties, or execute actions.

Amounts are bounded planning placeholders. Identifiers are deterministic pseudonymous references, and private customer, company, account, wallet, credential, document, endpoint, URL, and message fields are omitted.

Any live metering, billing, payment, settlement, network, storage, or commercial use requires technical, security, privacy, legal, tax, payment-provider, and commercial review plus Gareth final approval.
