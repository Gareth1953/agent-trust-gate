# Agent Market Radar Methodology

## Purpose

This document defines the local-only advisory method for the ATG Strategic
Foresight Layer. It is manual-input friendly and does not fetch, scrape, monitor,
or contact live sources.

## Method

1. Watch manually supplied notes, reviewer comments, public observations, and
   locally entered research summaries.
2. Analyse whether the signal affects GatePass, ProofPackage, VerificationContract,
   tool-call wrappers, scorecards, reviewer demos, pre-settlement blocking, or
   future protocol readiness.
3. Compare the signal against current local capabilities.
4. Recommend a possible future mission only if the signal is relevant.
5. Require Gareth approval before any build mission exists.

The workflow is:

Watch -> Analyse -> Compare -> Recommend -> Gareth approves -> Dave creates build mission -> Codex implements locally

## Boundaries

- No live scraping.
- No automatic monitoring.
- No autonomous outreach.
- No product changes.
- No code changes.
- No automatic roadmap changes.
- No prediction guarantee.
- No production readiness claim.

## Signal Categories

Signals may be grouped as AI agent frameworks, tool-calling patterns, developer
wrapper patterns, MCP-style protocols, agent-to-agent protocols, AI payment
protocols, machine-to-machine payment signals, pre-settlement trust signals,
agent security/governance competitors, enterprise agent adoption, developer
integration trends, AGI and agent-risk signals, quantum computing signals,
post-quantum security signals, standards/regulation signals, and
reviewer/developer feedback signals.

## Scoring Approach

Each signal receives local illustrative advisory scores for relevance to
GatePass, urgency, commercial impact, technical feasibility, reviewer credibility
impact, safety risk, implementation complexity, and dependency risk.

A high score means "consider a future mission." It does not mean "build
automatically."

## Gareth Approval Checkpoint

Any recommended future mission remains dormant until Gareth approves it. Dave
then turns the approved idea into a safe mission prompt. Codex implements only
inside that approved local mission.
