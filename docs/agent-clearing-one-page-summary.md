# Agent Clearing House + RefusalGraph

**Draft-only. Not published. Not a live website.**

> Before an agent says yes, it can check who already said no - and why.

## One-Sentence Explanation

Agent Clearing House is a local draft pre-action clearing and receipt layer that uses RefusalGraph caution signals to help an agent decide whether to accept, limit, refuse, or require stronger proof before a risky request.

## Problem

Agents will ask other agents and systems to perform work, access data, call tools, publish, purchase, and transact. A message or payment instruction can be delivered correctly and still be unsafe, unclear, or unauthorized.

## Solution

Agent Clearing House evaluates the request before action. RefusalGraph adds privacy-minimized patterns from similar actions that were blocked, refused, limited, or missing evidence and approval.

## How The Demo Works

1. A local placeholder agent declares a request.
2. Local RefusalGraph signals return a caution level.
3. A clearing decision accepts with limits, refuses, blocks, or requests proof and human approval.
4. An unpersisted local receipt records the result.
5. Local verification-readiness checks the receipt structure and safety flags.
6. A non-billable fee placeholder shows a possible future verification checkpoint.

No step executes the requested action.

## Why RefusalGraph Is Unique

Most systems focus on successful transactions or agent communication. RefusalGraph captures what was rejected and why, without copying raw private records. That negative trust intelligence can help future agents avoid repeating known risky patterns.

Agent Clearing House does not replace payment rails or agent communication protocols. It provides a local draft clearing, receipt, and refusal-intelligence layer before risky agent-to-agent actions.
