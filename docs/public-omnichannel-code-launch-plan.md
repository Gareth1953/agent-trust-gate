# Public Omnichannel Code Launch Plan

Agent Trust Gate™ should not be launched as a single-channel repository only. It should be presented as a code-readable trust layer with multiple safe discovery surfaces that all point back to one shared local Trust Gate core.

Omnichannel is an architectural direction, not a deployment mandate. It does not require launching every surface at once, creating separate products per channel, or activating a live system.

## First public launch surface: code

- GitHub/public repository
- README
- documentation
- static manifest
- schemas
- synthetic examples
- local CLI proof

These assets let developers inspect the rules, receipts, safety boundaries, and deterministic proof without connecting to a service.

## Later candidate surfaces

- static developer landing page
- agent-readable capability discovery
- package/npm readiness
- MCP/A2A-style adapter
- Agent Update Consortium bridge
- controlled sandbox

These later surfaces are not active today. Package publication, MCP/A2A integration, an AUC bridge, a hosted sandbox, live payments, settlement execution, and external agent contact all require separate review and approval.

AUC is not integrated. Agent Contact System is not integrated. The project remains local-first and `local_demo_only`.

**No mandate. No evidence. No verified intent. No signed gate pass. No settlement.**
