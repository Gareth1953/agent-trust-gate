# Public Positioning Claims Boundary

## Purpose

This document defines the public claims boundary for Agent Trust Gate™ after
P3-M140A.

## Allowed claims

- Local-first proof-of-concept.
- GatePass is a scoped proof primitive.
- GatePass is scoped, time-bound, and action-specific.
- GatePass provides a common, machine-readable format for expressing authority,
  mandate, scope, freshness, and evidence.
- Local deterministic demos.
- Local GatePass create / verify / reject lifecycle.
- Local adversarial scorecard with local illustrative timing.
- Local mock tool gating.
- Developer wrapper example.
- One-command reviewer kit.
- GatePass proof vocabulary and GatePass claims vocabulary as supporting
  material.

## Disallowed claims

- Production ready.
- Production middleware.
- Production-grade crypto.
- Certified secure.
- Legal, compliance, or security guarantee.
- Proven safe.
- Guaranteed trust.
- Real payment readiness.
- Real settlement readiness.
- Real-world benchmark performance.
- Complete adversarial coverage.
- Automatic paid-pilot acceptance.
- Automatic access after payment.
- Live API, hosted service, or external integration readiness.
- Real tool execution or action execution.

## Required public wording

Use restrained wording:

- "GatePass is a scoped, time-bound, action-specific proof primitive for agent
  actions."
- "No signed GatePass. No settlement."
- "Do not trust the agent. Trust the GatePass."
- "No proof. No permission."
- "No mandate. No action."

Avoid overclaiming:

- Do not say the agent is proven safe.
- Do not say GatePass guarantees trust.
- Do not say the demo is a production benchmark.
- Do not say the repo provides security certification.
