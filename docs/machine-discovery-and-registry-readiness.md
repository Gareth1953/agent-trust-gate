## Passive Machine Discovery

Passive machine discovery means publishing clear static metadata that can be read by humans and tools. It is safer than bot messaging or autonomous outreach because the repository waits to be discovered and reviewed; it does not contact agents, systems, buyers, competitors, marketplaces, developers, or reviewers.

The route is:

1. A human, search system, or AI assistant reads the repository.
2. The canonical discovery record explains current capabilities and inactive integrations.
3. A reviewer runs `npm run demo:reviewer-kit`.
4. A developer inspects GatePass, schemas, examples, and local wrappers.
5. A genuine prospect reads the Paid Evaluation Pilot docs and contacts Gareth manually.
