# Code-First Contact Model

Gareth's operating principle is code-readable contact first. Any future discovery by agents, systems, developers, or integration partners should begin with inspectable static assets:

- public repository
- manifest
- schemas
- documentation
- synthetic examples
- local CLI proof
- agent-readable capability statement
- static developer page, if separately approved later

This approach lets a reviewer understand the product and safety boundary without a live connection or sales process.

No phone calls. No manual sales chasing. No automated outreach. No live agent contact yet.

P3-M105 does not contact developers, systems, partners, agents, AUC, or Agent Contact System. AUC is not integrated. Agent Contact System is not integrated. The model is local-first guidance, not an active contact channel.

**No mandate. No evidence. No verified intent. No signed gate pass. No settlement.**
