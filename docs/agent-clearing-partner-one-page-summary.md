# Agent Clearing House + RefusalGraph

**Draft-only. Not externally shared. Not published.**

## One-Sentence Explanation

Agent Clearing House is a proposed pre-action clearing, refusal-intelligence, receipt, and verification layer for risky agent-to-agent requests.

## Problem

Agents can communicate and pay without proving that the requested action is authorized, adequately evidenced, within policy, or safe enough to proceed.

## Solution

The clearing layer evaluates declared intent, evidence, approval, identity state, payment intent, risk, and local refusal patterns before returning a machine-readable decision and receipt.

## Unique Advantage

Before an agent says yes, it can check who already said no - and why.

RefusalGraph adds privacy-minimized negative trust intelligence from blocked, refused, limited, evidence-missing, approval-required, and high-risk requests.

## Current Demo

The local demo composes refusal lookup, caution, decision, receipt, local verification readiness, a fee placeholder, CLI, readable report, and draft narrative. Nothing is networked or executed.

## Future Fee Model

Possible future fee points include clearing checks, receipt verification, and refusal lookups. They remain unvalidated hypotheses; no billable event, billing, payment, settlement, or machine fee is enabled.

## Why It Is Different

Agent Clearing House does not replace A2A, MCP, AP2, x402, Stripe, Coinbase, Mastercard, or other communication/payment systems. It provides a clearing, refusal-intelligence, receipt, and verification layer before risky agent-to-agent actions proceed.

## What Remains Disabled

External sharing, publication, outreach, email, CRM, tracking, analytics, deployment, signup, contact, billing, payment, settlement, fees, webhooks, third-party connections, public interfaces, networks, live verification, live metering, private-data export, automatic purchase, and execution are disabled.
