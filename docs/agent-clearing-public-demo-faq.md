# Agent Clearing Public Demo FAQ

**Status: draft-only, local-only, and unpublished.**

## Is this live?

No. It is a local demonstration and draft narrative. No public page, hosted service, or network is active.

## Does it connect agents today?

No. The demo composes local in-memory functions using supplied placeholder input. Agent-to-agent networking remains disabled.

## Does it move money?

No. It does not transfer, hold, authorize, or settle funds.

## Does it charge fees?

No. Fee fields are placeholders only. No billable event, billing record, payment, settlement, or machine fee is created.

## Does it replace payment systems?

No. Payment rails move value. Agent Clearing House is a proposed pre-action clearing, refusal-intelligence, and receipt layer before a system accepts a risky request.

## Does it replace A2A or MCP?

No. A2A-style protocols and MCP-style tools support agent communication and capabilities. Agent Clearing House evaluates trust conditions around a proposed action; it does not replace the communication protocol.

## What is RefusalGraph?

RefusalGraph is a draft machine-readable layer for privacy-minimized patterns from blocked, refused, limited, evidence-missing, approval-required, and high-risk actions. It helps explain what was rejected and what stronger proof may be needed.

## What is a clearing receipt?

A clearing receipt is a local draft record of a decision, caution level, safe reasons, evidence and approval state, and required next steps. It does not prove external truth or authorize execution.

## Why would agents need this?

Agents need a way to check more than whether a request can be delivered or paid for. They need to know whether it should be accepted, limited, refused, or escalated to a human.

## Does it use private customer data?

No real customer data is used in this draft. Outputs use normalized categories, reason codes, pseudonymous references, hashes, and control states. Raw identities, accounts, documents, credentials, and message content are excluded.

## Can it execute actions?

No. It returns local draft decisions, receipts, verification-readiness results, reports, and fee placeholders. It does not perform the proposed action.

## Can it be used commercially now?

No. Commercial use, live verification, fees, billing, payment, settlement, publishing, deployment, tracking, analytics, signup, outreach, and external connections remain disabled.

Gareth provides final approval. Publication also requires explicit publication approval after technical, security, privacy, legal, and commercial review.
