# Pre-launch Checklist

- [ ] Explicit Gareth approval before public launch or distribution
- [ ] Public repository and licensing decision
- [ ] Public developer documentation review
- [ ] Hosted sandbox or local-only launch decision
- [ ] Production security and abuse-prevention review
- [ ] Legal terms, privacy policy, and claims review
- [ ] Monitoring and incident-response review
- [ ] Support owner, contact, scope, and escalation process
- [ ] Pricing and plan decision without unsupported payment claims
- [ ] Public changelog, release, versioning, and package policy
- [ ] Brand and domain decision
- [ ] Analytics/tracking privacy decision before adding any measurement
- [ ] Payment provider review before paid launch
- [ ] Separate explicit approval before any automatic purchase capability

Until every relevant gate is complete, do not publish docs or source, expose an API, send outreach, run ads, add tracking, collect signups, process payments, or enable automatic purchase.
