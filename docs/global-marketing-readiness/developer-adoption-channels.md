# Developer Adoption Channels

Potential future channels include a reviewed public repository, developer docs site, npm package, OpenAPI and MCP directories, agent marketplaces, technical articles, a local demo video, protected hosted sandbox, enterprise pilot pack, and direct partner integrations.

No channel is live. Nothing has been published, listed, posted, advertised, or deployed.

Each channel requires a specific review because its risks differ. A public repository needs secret and licensing review. A package needs versioning and support policy. A hosted sandbox needs production authentication, abuse controls, monitoring, incident response, and legal terms. Marketplace and partner listings need accurate capability boundaries and operational ownership.

Existing local assets are the public-developer documentation draft, quickstart, OpenAPI contract, Node and PowerShell wrappers, agent manifest, and MCP-style adapter. These support review; they do not make the product publicly available.
